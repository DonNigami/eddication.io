# 🔧 UserData Import ไม่สมบูรณ์ - คู่มือแก้ไข

## สรุปปัญหาที่พบบ่อย

### 1. ⚠️ Date/Time Conversion Issues
**ปัญหา**: ข้อมูลวันที่/เวลาจาก Google Sheets แปลงเป็น Supabase format ไม่ถูกต้อง

**สาเหตุ**:
- Excel serial dates ไม่ถูกแปลงเป็น JavaScript Date
- String dates ที่ format ไม่ standard ทำให้ parse ล้มเหลว
- Time format ไม่ตรงกับ PostgreSQL `time without time zone`

**วิธีแก้ไข**:
```javascript
// ✅ ใช้ function ใหม่ที่รองรับทุก format
function convertToDateString(value, rowNum, fieldName) {
  if (!value) return null;

  try {
    if (value instanceof Date) {
      return Utilities.formatDate(value, 'GMT', 'yyyy-MM-dd');
    } else if (typeof value === 'string') {
      const date = new Date(value);
      if (!isNaN(date.getTime())) {
        return Utilities.formatDate(date, 'GMT', 'yyyy-MM-dd');
      }
    } else if (typeof value === 'number') {
      // Excel serial date
      const date = new Date((value - 25569) * 86400 * 1000);
      return Utilities.formatDate(date, 'GMT', 'yyyy-MM-dd');
    }
  } catch (e) {
    Logger.log(`⚠️ Row ${rowNum}: Invalid ${fieldName}: ${value}`);
  }

  return null;
}
```

### 2. ⚠️ Missing Error Handling
**ปัญหา**: ไม่ทราบว่า row ไหน import ล้มเหลว

**วิธีแก้ไข**:
```javascript
// ✅ Log ทุก row ที่ error
const errors = [];

for (let rowIndex = 0; rowIndex < data.length; rowIndex++) {
  try {
    // ... process row ...
  } catch (error) {
    errors.push(`Row ${rowIndex + 2}: ${error.toString()}`);
    Logger.log(`❌ Error processing row ${rowIndex + 2}: ${error.toString()}`);
  }
}

// สรุป error ทั้งหมด
if (errors.length > 0) {
  Logger.log(`\n❌ Errors:\n${errors.join('\n')}`);
}
```

### 3. ⚠️ Batch Size Too Large
**ปัญหา**: Batch ขนาดใหญ่ทำให้ timeout หรือ fail

**วิธีแก้ไข**:
```javascript
// ✅ ลด batch size และเพิ่ม retry logic
const BATCH_SIZE = 100; // เดิม 1000

// เพิ่ม retry mechanism
let retryCount = 0;
const maxRetries = 3;

while (!success && retryCount < maxRetries) {
  try {
    const response = insertToSupabase(tableName, batch);
    // ... handle response ...
  } catch (error) {
    if (retryCount < maxRetries - 1) {
      Utilities.sleep(1000 * (retryCount + 1)); // Exponential backoff
      retryCount++;
    }
  }
}
```

### 4. ⚠️ Header Mapping Issues
**ปัญหา**: Column headers ใน Sheets ไม่ตรงกับ code

**วิธีตรวจสอบ**:
```javascript
// รัน diagnostic เพื่อดูว่า column ไหนยังไม่ถูก map
function diagnoseUserDataImport() {
  const headers = sheet.getRange(1, 1, 1, lastCol).getValues()[0];

  headers.forEach((header, idx) => {
    const h = header.toString().trim().toLowerCase();
    const isMapped = isHeaderMapped(h);

    if (!isMapped && h) {
      Logger.log(`⚠️ Column "${header}" [${idx + 1}] not mapped`);
    }
  });
}
```

## 🎯 Common Issues & Solutions

### Issue: "registration_date is not a valid date"
**Cause**: ข้อมูลใน Sheets เป็น format ที่ JavaScript ไม่เข้าใจ

**Solution**:
1. เปลี่ยน format ใน Sheets: **Format** > **Number** > **Date**
2. หรือใช้ formula: `=TO_DATE(A2)` ถ้าข้อมูลเป็น text

---

### Issue: "user_id is required"
**Cause**: บาง rows ไม่มี user_id

**Solution**:
```javascript
// ตรวจสอบใน Sheets
=COUNTBLANK(A2:A)  // สมมติ user_id อยู่ column A

// Filter rows ที่ไม่มี user_id
=FILTER(A2:Z, A2:A="")
```

---

### Issue: "Import timeout"
**Cause**: ข้อมูลมากเกินไป หรือ Supabase response ช้า

**Solution**:
```javascript
// ลด batch size
const BATCH_SIZE = 50; // ลดจาก 100

// เพิ่ม sleep ระหว่าง batches
Utilities.sleep(500);
```

---

### Issue: "Duplicate key value violates unique constraint"
**Cause**: user_id ซ้ำกัน

**Solution**:
```javascript
// ตรวจสอบใน Supabase
SELECT user_id, COUNT(*)
FROM userdata
GROUP BY user_id
HAVING COUNT(*) > 1;

// ลบ duplicates
DELETE FROM userdata
WHERE id NOT IN (
  SELECT MIN(id)
  FROM userdata
  GROUP BY user_id
);
```

## 🎓 Best Practices

1. **เสมอใช้ Transaction logic**: Insert/Update แยกกัน
2. **Log ทุกอย่าง**: เผื่อ debug ภายหลัง
3. **Test ก่อนทำจริง**: ใช้ sample data ทดสอบก่อน

5. **Batch size พอดี**: ไม่ใหญ่เกินไป (100-500 rows)
